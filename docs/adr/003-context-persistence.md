# ADR-003: Хранение и восстановление контекста при смене аккаунта

**Статус:** Принято  
**Дата:** 2025

## Контекст

При смене аккаунта (исчерпан лимит) новый аккаунт не знает историю диалога.
Нужно передать контекст без потери.

## Решение

Двухуровневое хранение:
- **Redis** (TTL 24h) — быстрый кэш активной сессии
- **PostgreSQL** — долгосрочное хранение истории

При смене аккаунта `context-bridge`:
1. Загружает последние N сообщений из Redis/PostgreSQL
2. Если суммарный размер превышает лимит нового аккаунта — обрезает с начала
3. Возвращает готовый массив `messages[]` для передачи в API провайдера

## Ограничения

- Максимум токенов зависит от провайдера (передаётся в запросе)
- При обрезке истории теряется ранний контекст (отмечается флагом `truncated: true`)
- В будущем: сжатие через summary-промпт
