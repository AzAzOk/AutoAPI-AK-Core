# ADR-001: Структура монорепозитория

**Статус:** Принято  
**Дата:** 2025

## Контекст

Проект состоит из 5 микросервисов на двух языках (Go и Node.js).
Нужно решить: монорепо или несколько репозиториев.

## Решение

Монорепо с разделением по папкам `services/`.

## Причины

- Единый CI/CD пайплайн
- Общие типы и утилиты в `packages/`
- Проще синхронизировать изменения между двумя разработчиками
- Один `.gitignore`, один `Makefile` для всего

## Последствия

- Нужна умная CI-фильтрация (запускать только затронутые сервисы)
- Go и Node живут в одном репо — нужен общий `.gitignore` для обоих
