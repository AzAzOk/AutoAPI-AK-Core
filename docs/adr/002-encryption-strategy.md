# ADR-002: Шифрование API-ключей аккаунтов

**Статус:** Принято  
**Дата:** 2025

## Контекст

Пользователи хранят свои API-ключи от OpenAI, Claude и других сервисов.
Эти данные критически чувствительны.

## Решение

AES-256-GCM симметричное шифрование.
Мастер-ключ хранится в переменной окружения `VAULT_MASTER_KEY`.
Ключ никогда не логируется и не попадает в ответы API.

## Схема

1. Пользователь передаёт API-ключ через HTTPS
2. `account-vault` шифрует: `encrypt(apiKey, VAULT_MASTER_KEY)`
3. Зашифрованный blob сохраняется в PostgreSQL
4. При выдаче `router` расшифровывает на лету, использует в памяти
5. В логах и ответах API — только `account_id`, никогда сам ключ

## Последствия

- При утечке БД без `VAULT_MASTER_KEY` ключи не расшифровать
- `VAULT_MASTER_KEY` нужно ротировать вручную (добавить в roadmap)
- Нельзя индексировать зашифрованное поле — поиск только по `account_id`
